# Workflow Etiquette

Use the smallest workflow that carries the requested authority and proof. Research
is read-only. Normal Requests are bounded and reversible. Material work requires
one accepted outcome contract.

Before non-trivial implementation, the main agent runs Product, Engineering,
Design, Analytics, QA, and Performance lenses. Show material recommendations;
never adopt them silently. A lens may be `Not applicable` with a reason. Ask only
when a choice changes behavior, scope, authority, risk, cost, or external effects.

Plans are ephemeral. Preserve unrelated work and use subagents only when AJS asks.
Commit a coherent candidate before exact-revision Verify.

Accepted scope authorizes routine implementation, verification, commit, push, PR,
direct merge, declared deployment, and monitoring. Explicit narrower limits win.
Stop for unexpected destructive, credential, migration, privacy, security,
data-loss, or production effects.

Report implemented, committed, pushed, merged, deployed, and production-proved
states separately. Surface changed judgment, risk, blockers, evidence limits, and
user decisions; keep routine mechanics quiet.
