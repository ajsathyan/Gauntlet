# Skill Quality Bar

A skill earns context by changing a decision, authority boundary, safety outcome,
or behavioral proof.

Every retained skill states its trigger, job, output, completion condition,
negative case, proof layer, and authority. Keep each meaning in one authoritative
place. Put universal policy in the router and procedures in triggered skills.

Use deterministic code for mechanical facts. Text presence, populated fields,
receipts, and self-reports are structural evidence only. Prefer the cheapest
observable oracle that rejects a plausible wrong implementation.

Subagents are opt-in for AJS. Do not add profile selection, token audits, durable
queues, or role handoffs. Extra documents, ledgers, specialists, and exact-design
contracts are conditional on concrete consequence or repeated failure.

Remove stale examples, aliases, snapshots, compatibility paths, and generated
context that no retained caller needs. Git history preserves retired material.
